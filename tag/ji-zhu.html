记本次配置Pelican博客

2024年了,本来准备在自己电脑上搭个博客的,但是捣鼓了一圈端口啊域名解析啊什么的之后我还是

果断放弃

为什么呢?

原因有三:

  1. 没有可以在Cloudfare免费cdn用的域名,而家里的上行流量是远远不够的!

    专买一个怎么想都不值
    唯一可以的eu.org审核还没过
    还不是心疼钱

  2. GitHub Pages可以免费服务一个静态网页,虽然一个账号只有一个,但是有总比没有好
    当然网上也有freehosting和wix之类,据说效果不好就没再去深究
    中途倒是扫到过一个社区志愿的网页服务,挺感人,不过......一下就过去了

  3. "刚好"看到了 这份文档 和 Pelican官方文档 ,万事俱备只欠东风!

具体流程那两个教程里都有,我就不啰嗦了好吧
有几点注意提一下

  • GitHub用户名不分大小写.用creepydog.github.io和CreepyDog.github.io除了显示外没有任何区别!都是指向一个用户的.
  • 虽然我最后是git push由pelican content生成的output文件夹,但是建议刚刚开始的直接用 官方文档里Tips栏的GitHub Action方法 .这样更加方便,同时节约本地资源,你甚至只需要git环境就能开写!

写在最后的感受

最终搞完的感觉就是,
像git这种工具
完全没必要刻意去学
需要用的时候不用一天就能上手

事实上,我现在还对git里很多如merge,fetch之类的命令完全不知道怎么使用! 看文档也不大能明白.
不过即便仅仅会了以下几个git命令目前也够了

git init
git checkout -b main
git config user.name "CreepyDog"
git config user.email *我邮箱*
git remote add origin *项目链接*
git add .
git commit -m *message*
git push -u origin main

为啥会这样?这次Pelican配置就用到这几个!
